Checklist de Code Review

1. Estrutura e Organização

  • Nomes descritivos para variáveis, métodos e classes
  • Cada classe ou método tem uma única função
  • O código está modularizado e reutilizável

2. Legibilidade e Clareza

  • Indentação e formatação consistentes
  • Comentários úteis para trechos complexos
  • Evita duplicação de código (DRY)

3. Lógica e Fluxo de Controle

  • Estruturas de controle (if, else, loops) claras e corretas
  • Tratamento adequado de exceções

4. Segurança e Validações

  • Validação de entradas do usuário
  • Dados sensíveis são tratados com segurança

5. Testes

  • Cobertura de testes para funcionalidades principais
  • Testes cobrem situações de erro e entradas inválidas
